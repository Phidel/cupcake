CUPCAKE

Code editor written in pure python

Open source

I wrote Cupcake for learning purposes. Since I hightly enjoy using python, I thought about making my code editor in it too. Cupcake uses python's tkinter library.
Although it is not a full-fledged code editor, it is a good start to get more into tkinter, and explore its capabilities. Since tk is considered comparatively old to the frameworks used nowadays, it surely is not the best choice out there to write something like this. But I wanted to test it out: what tk can do with all it's got. Of course all the components used are not part of the tkinter library, I had to create many of them my own. Even the holy autocompletion, without which a code editor is not a code editor nowadays, I had to make from the very scratch, a literal toplevel window.

Capabilities

Coding experience with cupcake is good enough. It's not as good as VSCode or something like that, but still manages to get a fairly good review from users. Cupcake offers the following features. The behavior of the components may not be as expected, but hey bug reports can help me fix that issue.

  • Autocompletion of keywords and words used in the script
  • Syntax highlighting of keywords, numbers, strings, etc.
  • Minimap for quick navigation through code
  • Extendable language support

Have a Cupcake!
